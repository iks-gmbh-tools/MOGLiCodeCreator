VelocityClassBasedFileMaker    Property List     v1.6.1

In the header of the main templates* for input artefacts* of this plugin the following artefact properties can be defined:

1. TargetDir – Folder in your file system where to write the generated content. If not defined, output will only be written in the plugin’s output directory. Use “<applicationRootDir>” (without quotes) to address the directory where the start script startMOGLiCodeCreator is located.

2. TargetFileName – Name of file in which the generated content is saved.

3. CreateNew – If ‘true’, both the target folder (if not existing) is created and the content of the existing target file is replaced. If ‘false’ and the target folder not existing, an error will be reported. If ‘false’ and the target folder does exist, the target file will be created if not existing. If ‘false’ and the target file does exist in the target folder, the existing file will remain unchanged. Default is false.

4. NameOfValidModel – Determines that the template is only applied if the name of the current model (provided by the StandardModelProvider) is equal to the value of this template property. If not provided, the template will be never applied.

5. OutputEncodingFormat – Encoding applied when generating the artefact’s output file(s). Default is UTF-8. Since 1.1.0

6. SkipGeneration – If ‘true’, code supposed to be created by this template is not generated. Default is false. Can be used to suppress the generation of a specific artefact. The following use cases are applicable: a)  @SkipGeneration $isDatabaseRelevant, b) @SkipGeneration NOT $isDatabaseRelevant, c) @SkipGeneration $type == DomainObject, d) @SkipGeneration $type != DatabaseObject. Since 1.2.0 – c, d Since 1.6.0

7. ReplaceToNumberSign – Any string that will be replaced by the # symbol after Velocity has merged model and template and before the output file is created. Since Velocity interprets all number signs and its following chars as comment, these comments are removed during the Velocity merge of model and template. Thus, no number sign can get through this merge. As a workaround to get number sign in the final output, a number sign placeholder can be defined by this artefact property. Since 1.6.0

8. CheckMissingMetaInfos – If ‘false’, metainfo used in template files but not defined in the model data are ignored. If ‘true’, an error report is created in case a metainfo could not be resolved. Default is true. Since 1.6.1

 

* An input artefact is a subdirectory in <applicationRootDir>/input/VelocityClassBasedFileMaker that contains a main template. A main template’s name contains ‘Main’. A main template may call other templates. All templates have the file extension tpl. Input artefacts of the VelocityClassBasedFileMaker must have exactly one main template.

To study a simple example using the VelocityClassBasedFileMaker see this Hello World example.

To study a complex example using the VelocityClassBasedFileMaker, execute MOGLiCC with “modelfile= MavenProjectWithJavaDomainModel.txt” as content of <applicationRootDir>/input/StandardModelProvider/_model.properties. This will merge the data of the model file with the MOGLiJavaBean-templates which you can find in the directory <applicationRootDir>/input/ VelocityClassBasedFileMaker.
