The MOGLi Code Creator

 

Just another code generator? – NO! – It’s THE Lightweight autogeneration tool!

 

 

 

Getting started

 

1.    Have a Java 1.6 (or higher) runtime environment (JRE) installed.

 

2.    Provide this JRE to MOGLiCC

       a)  either by setting the JAVA_HOME system variable

       b)  or by editing the start batch or shell script file and defining the “ScriptJavaHomeBin” variable.

 

3.    Depending on your operating system, start the batch or shell script. This will execute the MOGLiCC Life Demo.

 

Done.

 

After executing MOGLiCC, you find much more information about MOGLiCC here: help/MOGLi Code Creator.htm.

 

In addition,

 

4.    you may wish to integrate the execution of MOGLiCC in your IDE. For eclipse, this is possible by the “External Tool Configuration”.

 

5.    you may wish to use your IDE as editor for model and template files, create a new workspace project that includes the MOGLiCC root directory (this is the directory where you unzipped the MOGLiCC sources and where this readme file is located).

 

 

 

 

Now what?

 

1.    After executing the Life Demo, a number of directories and files have been created  – all based on the default model and default templates extracted from the sources. You may study the report directory. The reports contain detailed information, what MOGLiCC has done.

 

2.    The Life Demo uses input artefacts of the MOGLiCC Java Bean Group. You may study these input data (see input directory).

 

3.    The Life Demo creates output artefacts twice: firstly, in the output directory and secondly in the defined target directory called MOGLiCC_JavaBeanProject. You may study these generation results.

 

4.    The Life Demo generates the target directory “MOGLiCC_JavaBeanProject” which contains a Maven Java project ready to be imported by an IDE capable of doing so. The IDE allows studying the output artefacts more conveniently. The sources of this project should compile and the Junit tests are ready to be executed – assumingly without test failures. The functioning of the generating results may demonstrate you the power of both MOGLiCC as tool and its default functionality of the JavaBeanGroup.

 

 

 

How can I manipulate?

 

1.    Open the model file “input\StandardModelProvider\MOGLiCC_JavaBeanModel.txt” and start manipulating the classes, attributes and metainfos. Modify, add and remove as you wish.

 

2.    Save the model file, execute MOGLiCC again and study the generation results. See also How does MOGLiCC work?

 

3.    You may also manipulate the template files. Without experience on the Velocity Template Language this is more difficult. However, if used to Java syntax, you will quickly help yourself.

 

4.    Have a look at the Hello World Simple Example.
