VelocityModelBasedTreeBuilder    Property List     v1.6.1

In the artefact propertiey file* of input artefacts* of this plugin the following artefact properties can be defined (italic properties are specific to this generator):

1. TargetDir – Folder in which the generated tree is written. If not defined, output will only be written in the plugin’s output directory. Use ‘<applicationRootDir>” (without quotes) to address the directory where the start script startMOGLiCodeCreator is located.

2. RootName – Name of the top folder created in the target directory. If not set, the artifact name is used as default.

3. CreateNew – If ‘false’, a non-existing target directory produces an error, but a non-existing output artefact is created (when the target directory exists) and an already existing target directory (defined by the ‘targetDir’ property) will not be modified. If ‘true’, the modifications depend on the CleanTarget and PreserveFiles properties. If not set, default ‘false’ is used.

4. NameOfValidModel – If equal to the name of the model provided by the ModelProvider the artifact is generated. If not set, the artifact is generated for all models.

5. OutputEncodingFormat – Encoding applied when replacing placeholders within text files. If not set, ‘UTF-8’ Is used as default.

6. SkipGeneration – If ‘true’, code supposed to be created by this template is not generated. Default is false. Can be used to suppress the generation of a specific artefact. The following use cases are applicable: a)  @SkipGeneration $isDatabaseRelevant, b) @SkipGeneration NOT $isDatabaseRelevant, c) @SkipGeneration $type == DomainObject, d) @SkipGeneration $type != DatabaseObject. Since 1.2.0 – c, d Since 1.6.0

7. CheckMissingMetaInfos – If ‘false’, metainfo used in template files but not defined in the model data are ignored. If ‘true’, an error report is created in case a metainfo could not be resolved. Default is true. Since 1.6.1

8. PreserveFiles – If ‘true’, already existing files in the target directory (defined by the ‘targetDir’ property) will not be overwritten. If not set, default ‘false’ is used. Note, that this property cannot be used in combination with CleanTarget.

9. CleanTarget – If ‘true’, existing target directory (defined by the ‘targetDir’ property) will be deleted before generation. This helps to get rid of old files which are no more used. If not set, default ‘false’ is used. Note, that this property cannot be used in combination with PreserveFiles.

10. Exclude – Comma separated list of files or directories that do not belong to the artifact and that are ignored when copying the file structure. The artifact.properties file is automatically excluded. Useful for e.g. ignoring data only needed by your version control system.

11. ReplaceIn – Defines a replacement of a placeholder within a text file. Placeholders in the defined file are replaced by the text defined here. This text cannot contain line breaks.

12. RenameFile – Defines a file renaming that causes a file from the artifact’s input directory to have a different name in the artifact’s output and target directory.

13. RenameDir – Defines a file renaming that causes a directory from the artifact’s input directory to have a different name in the artifact’s output and target directory. In this way parts of the file path can be replaced, e.g. ‘placeholderPath” by ‘com.iksgmbh.example”.

 

* An input artefact is a subdirectory in <applicationRootDir>/input/VelocityModelBasedTreeBuilder that contains a file named artefact.properties which contains information that is for the other two generator types contained in the template header. All files and folders found in the input artefact’s subdirectory belong to the tree that this generator is going to build for this input artefact. A file or folder may be excluded from the input artefact (see Exclude property above). The artefact.properties is by default not part of the input artefact.

To study a complex example using the VelocityModelBasedTreeBuilder, execute MOGLiCC with ‘modelfile=MavenProjectWithJavaDomainModel.txt” as content of <applicationRootDir>/input/StandardModelProvider/_model.properties. This will merge the data of the model file with the MOGLiCC_JavaBeanProject-template which you can find in the directory <applicationRootDir>/input/VelocityModelBasedTreeBuilder.
