The model of the StandardModelProvider

A model for the StandardModelProvider is constructed out of classes, attributes and metainfos. The MOGLiCC meta model consists of the three hierarchy levels model, class and attribute. A model file must contain exactly one model. The name of a class (for a java class it should also contain the qualified path) must be unique within the model. The name of an attribute must be unique for its class. For example:

model CarRace

    class VW.Lupo

        attribute engine

 

Each element (model, class or attribute) is defined by its name - the only concretely predefined data value. The data of each element in each hierarchy level can be enriched by MetaInfo elements in any number. A MetaInfo element consists of a non-unique name and a value. For example:

model CarRace

  metainfo type CityTour

    class VW.Lupo

      metainfo weight 1000kg

        attribute engine

          metainfo power 50 PS

 

For repreatedly used values (such as paths to domain objects) variables can be defined. A variable consists of a key-value-pair. The key can be used as placeholder: <<variableKey>>. When the model file is parsed all placeholders are replaced by its variable value. Variables can be defined anywhere in the model file, e.g. at the end of the model file. Variables values may contain placeholders of other variables, which are mentioned above in the model file.

variable vw Volkswagen

model CarRace

    class <<vw>>.Lupo

    class <<vw>>.Passat

 

For an example with more detailed information about writing model files, execute the MOGLiCC without an input directory. This will make MOGLiCC to extract its default data. Then have a look into the model file MOGLiCC_JavaBeanModel.txt. Alternatively, study the Hello-World-Examples.
